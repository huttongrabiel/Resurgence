Detailed Description

Represents a 2D global point cloud map.

As the robot moves around, it will store information about its environment for mapping purposes.

Creates a new GlobalMap object, for use with building global lidar point clouds.

Parameters
areaSizeThe side length of the square represented by this global map. The units don't matter, as long as they're consistent with the data.
scanStrideIndicates the amount of points per scan to ignore. The number of points kept from each scan is 1/scanStride, rounded down.
maxIterMaximum iterations allowed for fitting new samples to the map
relErrChangeThreshMinimum relative error change threshold for fitting new samples to the map. Iteration will terminate early if the relative change in error drops below this amount.

Register the given points into the global map.

Parameters
robotTrfThe transform of the robot pose.
pointsThe lidar sample point cloud, in the reference frame of the given robot transform.
overlapThe estimated overlap, in the range [0,1] between this sample and the global map. It's better to underestimate than overestimate. However, if it's too low then no meaningful features can be matched. Set to 0 to add points directly with no feature matching.

Checks if the given point has at least one point within the given distance from it.

Although it's functionally equivalent to checking the size of the vector returned by getPointsWithin(), it is much cheaper to call this method.

Parameters

distThe distance within which to search.
Returns
True iff there exists a point in the map, which may be the given point, within the given distance from the given point. False otherwise.
